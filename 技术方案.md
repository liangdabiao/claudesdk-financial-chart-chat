# FinancialChartChat — 技术方案

## 一、项目概述

基于 `claude-agent-sdk` 构建 AI 财经图表对话 Web 应用。用户通过自然语言查询 A 股上市公司财报数据，AI 自动获取数据并生成专业可视化图表（PNG）。

**核心能力**：说一句话 → AI 自动查数据 → 生成高清图表 → 在线预览

## 二、Skill 深度分析

### 2.1 原始技能文件

```
financial_chart_skill/
├── financial_charts.py   # 核心图表生成模块（6种图表类型）
├── demo.py               # 完整使用示例
├── requirements.txt      # matplotlib, numpy
└── README.md             # API文档
```

### 2.2 六种图表类型

| 函数 | 图表 | 用途 |
|------|------|------|
| `chart_revenue_profit()` | 双柱状图 | 营收+净利润年度对比 |
| `chart_margin_delivery()` | 折线+柱状 | 毛利率+交付量趋势 |
| `chart_quarterly_trend()` | 双Y轴 | 季度营收+毛利率趋势 |
| `chart_peer_comparison()` | 2×2子图 | 同行多指标对比 |
| `chart_pie_comparison()` | 双饼图 | 结构占比变化对比 |
| `chart_line_trend()` | 多线折线图 | 多公司/多指标趋势 |

### 2.3 关键技术点

- **中文字体修复**：`setup_chinese_font()` 解决 matplotlib 中文乱码（Windows 使用微软雅黑/黑体）
- **诗与星空配色**：专业财经配色方案（蓝/橙/绿/黄/红五色系统）
- **高清输出**：默认 dpi=200，白底保存

### 2.4 数据源扩展

原始 skill 仅包含图表生成，**不含数据获取**。根据项目说明，需要补充：
- **akshare**：Python 开源财经数据库，免费获取 A 股财报数据
- **东方财富 API**：备选数据源
- AI 通过 Bash 执行 Python 脚本获取数据

## 四、核心 Skill 设计

### 4.1 技能文件结构

```
.claude/skills/financial-charts/
├── SKILL.md              # 技能描述文档（AI读取此文件决定何时调用）
├── financial_charts.py   # 图表生成核心（来自原始 skill）
├── data_fetcher.py       # 数据获取模块（新增，基于 akshare）
└── requirements.txt      # Python 依赖
```

### 4.2 SKILL.md 核心逻辑

SKILL.md 定义 AI 的行为流程：

```
用户输入 → 意图识别 → 分支处理：

1. 查询财报（"查询贵州茅台2024年年报"）
   → data_fetcher.py 获取 5 年财务数据
   → financial_charts.py 生成 15 指标 × 2 类型 = 30 张图表
   → 输出到 charts/{公司名}/

2. 对比分析（"对比蔚来理想小鹏营收"）
   → data_fetcher.py 获取多公司数据
   → chart_peer_comparison() 生成对比图
   → 输出到 charts/{对比主题}/

3. 自定义图表（用户提供数据要求画图）
   → 直接调用 financial_charts.py 对应函数
   → 输出到 charts/custom/

4. 趋势分析（"贵州茅台营收趋势"）
   → data_fetcher.py 获取时间序列数据
   → chart_quarterly_trend() / chart_line_trend()
   → 输出到 charts/{公司名}/
```

### 4.3 data_fetcher.py 设计（新增）

基于 **akshare** 的 A 股数据获取模块：

```python
# 主要功能：
# 1. 公司名 → 股票代码 查询
# 2. 年报/季报/半年报 数据获取
# 3. 15 个关键财务指标提取
# 4. 同行业公司数据批量获取

# 关键指标：
# 资产负债表：货币资金、交易性金融资产、应收账款、存货、
#            短期借款、长期借款、合同负债
# 利润表：营业收入、归母净利润、销售费用、研发费用、财务费用
# 现金流量表：经营性现金流、资本支出

# 输出格式：标准化 dict，直接喂给 financial_charts.py
```

## 五、前后端设计

### 5.1 服务端（server/index.ts）

```
端口：3015
路由：
  GET  /api/charts       → 扫描 charts/ 目录，返回文件树
  GET  /charts/*         → 静态图片文件服务
  POST /api/upload       → 文件上传（CSV/Excel 数据导入）
  WS   /ws               → WebSocket 实时通信

  - charts/ 目录扫描 → 返回 { name, path, type, children }[] 结构

  - MessageQueue + AgentSession 复用已验证的 SDK 集成模式
```

## 七、与 amazon-skills-chat 的关键差异

| 维度 | amazon-skills-chat | financial-chart-chat |
|------|--------------------|----------------------|
| Playwright | 需要（浏览器自动化） | **不需要** |
| Python 依赖 | 少量脚本 | matplotlib + numpy + akshare |
| 输出格式 | Markdown 报告 | **PNG 图表图片** |
| 预览方式 | 文本预览 | **图片预览** |
| 数据来源 | Amazon 网页抓取 | **akshare API** |
| 工具需求 | Bash + Playwright | **Bash（Python 执行）** |
| 前端侧栏 | 文本文件列表 | **图片缩略图列表** |

## 九、风险与对策

| 风险 | 对策 |
|------|------|
| akshare API 限速/不稳定 | 设置重试机制 + 备用数据源 |
| Windows 中文字体缺失 | `setup_chinese_font()` 适配微软雅黑/黑体 |
| 图表生成耗时（30张） | 逐张生成+流式反馈，不阻塞 |
| Python 环境依赖 | 启动时检测，缺失时提示安装命令 |
| SDK 进程冲突 | 使用独立 .claude 目录，CLAUDE_CONFIG_DIR 隔离 |
